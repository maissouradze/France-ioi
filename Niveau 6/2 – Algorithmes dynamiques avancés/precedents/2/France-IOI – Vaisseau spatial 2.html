Il s'agit du même problème que le 1, sauf qu'il faut maintenant renvoyer la séquence de mouvements qui permet d'obtenir le meilleur score.

Limites de temps et de mémoire (C++)

  • Temps : 0,25 s sur une machine à 1 GHz.
  • Mémoire : 2 000 ko.

Contraintes

Les contraintes sont identiques :

  • -1000 <= B <= 1000, où B est la valeur d'un bonus (ou malus).
  • 1 <= N <= 2000, où N est le nombre de lignes du jeu.
  • 1 <= M <= 50, où M est le nombre de colonnes du jeu.
  • 0 <= P < M, où P est la colonne où se trouve votre vaisseau au départ.

Entrée

  • La première ligne de l'entrée contient trois entiers :
    • N, le nombre de lignes.
    • M, le nombre de colonnes.
    • P, l'indice de la colonne de départ.
  • Chacune des N lignes suivantes contient M entiers, décrivant la valeur des bonus (ou malus) contenus dans le jeu. La ligne 2 représente la ligne du haut du jeu, et la ligne N+1 représente la ligne du bas du plateau, là où se trouve votre vaisseau au début du jeu.

Sortie

Vous devez écrire une ligne sur la sortie, contenant N-1 entiers, séparés par des espaces. Chaque entier représente un déplacement : -1 pour gauche, 0 pour aller tout droit, 1 pour aller à droite.

Il se peut qu'il y ait plusieurs possibilités; si à un moment on peut faire le meilleur score soit en allant tout droit soit en tournant, on ira tout droit. Si le choix est entre tourner à gauche ou tourner à droite, on ira à gauche.

Exemples

Exemple 1

entrée : 

4 3 1
3 -1 4
2 0 0
0 2 0
0 0 0

sortie : 

0 -1 0

Exemple 2

entrée : 

4 3 1
1 1 0
0 2 0
2 0 2
0 0 0

sortie : 

-1 1 0

Il faut retenir le chemin. Pour cela, à chaque fois que l'on détermine la valeur du maximum des trois cases d'au-dessus, on va en plus retenir quelle case contient le max, c'est-à-dire la valeur de "dep" qui atteint le maximum. Voici un premier jet, où origine est un tableau qui retient pour chaque case vers où il faut se diriger pour obtenir un score maximum. A là fin pour retrouver le chemin, il suffira de suivre les indications marquées dans le tableau "origine" pour retrouver le chemin.

{
   int score = meilleurScore(lin-1, col+dep);
   if (score > meilleur)
   {
      meilleur = score;
      origine[lin][col] = dep;
   }
dynMeilleurScore[lin][col] = jeu[lin][col] + meilleur;

Malheureusement ce code ne fait pas ce qu'on veut, car l'ordre de priorité dans le choix des directions n'est pas respecté. En effet, si les trois cases du dessus sont de même valeur, notre algo va retenir la direction dep = -1. Ce qu'on veut faire, c'est donc essayer les trois directions dans l'ordre suivant : dep=0, puis dep=-1, puis dep=+1. Comment coder cela facilement ? On va utiliser un tableau de trois valeurs égal à { 0, -1, +1 }, que l'on va nommer directions.

int nbDir = 3;
int directions[3] = { 0, -1, +1 };

Maintenant on fait une boucle sur les indices de directions, et on récupère dep = directions[idDir], et ça marche:

int meilleur = MOINS_INFINI;
for(int dir = 0; dir < nbDir; dir++)
{
   dep = directions[dir];
   int score = meilleurScore(lin-1, col+dep);
   if (score > meilleur)
   {
      meilleur = score;
      origine[lin][col] = dep;
   }
dynMeilleurScore[lin][col] = jeu[lin][col] + meilleur;

Maintenant, il faut retrouver le chemin. On l'a dit il suffit de suivre les flèches au cours des nbLin - 1 étapes. Voici le principe :

void afficheCheminDepuis(ligne, colonne)
   si on est à la première ligne, return;
   soit dep le déplacement à effectuer à la case (ligne, colonne)
   affiche la valeur de dep
   afficheCheminDepuis(ligne-1, colonne+dep)

Cela donne une fois codé une fonction que l'on appel sur (nbLin - 1, colDepart) pour commencer :

void afficheCheminDepuis(lin, col)
{
   if (lin == 0) return;
   printf("%d ", origine[lin][col]);
   afficheCheminDepuis(lin - 1, col + origine[lin][col]);
}

On peut remplacer cette fonction récursive par une boucle while très facilement:

void afficheChemin()
{
   int lin = nbLin - 1;
   int col = colDepart;
   while (lin > 0)
   {
      printf("%d ", origine[lin][col]);
      col += origine[lin][col];
      lin--;
   }
}

Encore mieux, comme on sait quelles sont les valeurs prises par lin, on peut faire une boucle for :

void afficheChemin()
{
   int col = colDepart;
   for (int lin = nbLin - 1; lin > 0; lin--)
   {
      printf("%d ", origine[lin][col]);
      col += origine[lin][col];
   }
}

Comparaison entre les deux versions :

  • 36 vraies lignes de code en récursif, contre 28 en itératif. Ce n'est donc pas le faible gain en nombre de lignes qui peut justifier l'utilisation de telle ou telle version.
  • La version récursive consomme un tableau en plus en mémoire, mais ceci n'est absolument pas génant tant qu'on est pas vraiment juste par rapport à la contrainte en mémoire. De plus avoir des tableaux séparés pour les données d'entrées et les données calculées aide à débugger, car on peut les afficher plus facilement. Donc c'est plutôt un bon point pour la version qui ne travaille pas en place dans le tableau jeu.
  • La version itérative nous à obligé à rajouter des colonnes, ce qui a eu pour conséquence de décaler les indices des colonnes, et cela à introduit quelques difficultées supplémentaires au niveau des boucles, et au niveau des accès aux valeurs finales. La difficultés amenant des bugs, c'est un mauvais point pour la version itérative sur ce sujet.

Conclusion : pour ce sujet, la version itérative n'apporte que des inconvénients.
